Dear managers: Stop creating your technical documentation in word!

Ladies and gentlemen, it’s time for a good old-fashioned rant about something that gets my blood boiling: Word documents. If you’re using them for technical documentation, you’re doing it wrong. Let me tell you why.

First of all, have you ever tried to format a Word document? It’s like trying to juggle flaming swords while riding a unicycle. You spend hours fiddling with margins, fonts, and line spacing, only to have it all go to hell when you try to print it out. And don’t even get me started on the nightmare that is adding images or tables.

But it’s not just the formatting that’s the problem. Word documents are also a nightmare to collaborate on. Every time you make a change, you have to save a new version, which means keeping track of different versions of the same document. It’s like trying to herd cats. And if you’re working on a team, forget about it. You’ll spend more time emailing different versions back and forth than actually getting any work done.

Storing word documents is a nightmare too — there is nothing worse then when you have a system on fire to find a share-point repository with 249 files with obscure (if not, downright unhelpful) names… You also need to consider the overhead of opening up these files. Each one may only take 5 seconds but this is nothing short of painful when you’re time poor!

Searching for specific content across multiple documents is borderline impossible!

Documentation is rarely considered a glamourous job (sorry fellow authors!), everything should be done to make it easier and lowing the barrier to entry. Needing to fire up a heavy-weight client like Microsoft Word is only going to creating more friction at the starting gates. Not to mention — word documents often encourage having multiple sources of truth which is the source of all evil!

Now, let’s talk about wikis. Ah, sweet, sweet wikis. The ultimate documentation tool. First of all, they’re collaborative. That means that you can have multiple people working on the same document simultaneously without having to worry about version control. You can even see who made what changes and when they made them. It’s like having your very own team of documentation elves.

And let’s not forget about how easy it is to format wikis. Need to add a table? Just click a button. Want to insert an image? Piece of cake. Want to add a link to another page? Just type it in. It’s so easy, a monkey could do it.

In conclusion, if you’re still using Word documents for technical documentation, you’re doing it wrong. Wikis are the way to go. They’re collaborative, easy to format, and they’ll save you a lot of time and headaches. So, put down that Word document and start using wikis. Your documentation will thank you for it.
